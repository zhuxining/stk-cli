# 信号策略参考

**何时读取**：当你需要解读扫描结果中的 `decision`（信号和强度）、`primary_signal`（触发来源）、`context`（辅助因子态度）、`risk`（风控边界）的**交易含义**时读取此文件。

**关联文件**：字段的数据结构和位置见 `references/output-schema.md`。本文解释各取值的**交易含义**和解读原则。

**核心分析原则**（详见末尾"分析原则"节）：
1. 先下结论：`decision.signal` + `strength` + `signal_status`
2. 验证主信号：触发来源，`bars_since_signal` 越小越新
3. 校验辅助因子：统计确认/冲突/风险/机会数量
4. 给执行边界：止损、目标位、风报比
5. `focus`（推荐/预警）才展开分析，`观察` 只统计不展开

---

## 信号类型

| signal     | 含义                                                 |
| ---------- | ---------------------------------------------------- |
| `趋势买入` | 趋势共振信号，完整日线确认                           |
| `趋势退出` | 趋势翻转信号，表示减仓/退出/风险预警，不表达做空建议 |
| `超卖修复` | 左侧修复信号                                         |
| `观察`     | 无效/过期/冲突/确认不足/风险收益比不足               |

退出类信号的 `risk.stop_loss` 是上方失效线，`risk.target_1`/`target_2` 是下行风险参考位。`risk.trailing_stop` 是当前 Supertrend 值，供动态止损参考。

## 信号强弱

- `推荐`：买入信号成立（趋势买入、超卖修复），基础条件全部满足
- `预警`：退出/风险信号成立（趋势退出），基础条件全部满足

无信号（strength = None）不进入 `focus`，不计入扫描关注。

## 有效窗口与触发条件

非对称策略——进场严格、出场灵敏：

| 方向 | 窗口 | 触发条件 |
|------|------|----------|
| 多方（推荐） | 3 根K线 | EMA 金叉 **AND** Supertrend 翻多同时出现 |
| 空方（预警） | 5 根K线 | EMA 死叉 **OR** Supertrend 翻空任一出现 |

bull_event_age = max(金叉年龄, 翻多年龄) — 取较晚事件
bear_event_age = min(死叉年龄, 翻空年龄) — 取较早事件

## 日线确认

扫描结果（信号、辅助因子、成交量、风控）只使用已确认的完整日线：

- 盘前和盘中沿用上一交易日
- 盘后超过确认缓冲时间后才纳入当天日线
- `last` / `change_pct` 只展示当前价格，不参与信号计算

## 实盘扫描（scan-live）

独立提醒模式，不改写日线 `decision`：

- 先用完整日线信号做底层过滤（`观察` 和过期信号不参与）
- 再用已完成 5m/15m K 线判断盘中触发

| live_signal | 条件                                                                 |
| ----------- | -------------------------------------------------------------------- |
| `实时跟随`  | 日线偏多 + 分钟收盘站上 VWAP 和 EMA20                                |
| `实时转弱`  | 日线偏多但分钟跌破 VWAP/EMA20/开盘区间低点；或日线退出信号下继续弱势 |
| `实时过热`  | 日线偏多 + 分钟 RSI 或相对 VWAP 偏离过热                             |

`risk_line` 是分钟触发失效线，`volume_ratio` 是分钟成交额对比（不等同于日线 `volume_ratio_5d`）。

## 辅助因子态度

`context.overall_bias` 取值：

| bias          | 含义                          | 对结论的影响           |
| ------------- | ----------------------------- | ---------------------- |
| `supportive`  | 辅助因子整体支持主方向        | 可强化结论             |
| `mixed`       | 信号可关注，但需等待继续确认  | 保持信号级别，提示等待 |
| `risky`       | 有过热/缩量/布林收窄/资金转弱 | 避免追高               |
| `conflicting` | 辅助因子明显冲突              | 降低措辞强度           |

默认扫描结果省略 `neutral` / `none` 因子，完整复盘时加 `--full-context`。

## 指标解读要点

### RSI

动量温度，不单独产生买卖建议。多头中 `50-70` 更健康，`>70` 偏追高风险，`<45` 动能不足。

### ADX

趋势强度提示。ADX `<20` 趋势偏弱信号易反复，`>=25` 趋势质量更好。不直接改变 `strength`。

### KDJ J

观察短期过热或钝化。极高 J 值是追涨风险，不直接否定趋势。

### MFI

结合方向解读：多头中强 MFI 是确认，空头中弱 MFI 是确认。极端超买/超卖优先作为风险或机会提示。

### 背离（divergence）

MACD histogram 背离。顶背离 = 趋势减速风险，底背离 = 左侧机会提示。两者都不能替代主信号。

### 布林带（boll）

`position_pct` 接近上轨 = 强势但追高风险增加，接近下轨 = 弱势或修复机会。`bandwidth_pct` 很低 = 波动收敛，等待放量选方向。

### 量价（volume_price）

`volume_ratio_5d` 基于完整日线成交额。放量同向 > 缩量同向，放量逆向列为风险。

### 缩量金叉预警

EMA 金叉触发时，自动检查量比（当前成交额 / 近 5 日均值）：
- 量比 < 0.8 → `reasons` 中追加 `缩量金叉 (量比 X.X)，突破有效性存疑`
- 不改变 `strength`，仅提供风险上下文

### EMA60 长周期趋势

EMA 金叉时检查收盘价与 EMA60 的关系：
- 收盘价 < EMA60 → `reasons` 中追加 `价格仍在 EMA60 (XXX.XX) 下方，长周期趋势偏空`
- 提示日线金叉可能只是熊市反弹，不改变 `strength`

### 分层出场

`RiskProfile` 提供两个目标位和一个动态止损参考：
- `target_1`（2×ATR）：保守目标，减仓参考
- `target_2`（3×ATR）：激进目标，清仓参考
- `trailing_stop`（当前 Supertrend）：后续交易日的动态止损参考位

### 市场状态

`MarketOverview.regime` 按区域输出 `trending` / `ranging` / `mixed`。Agent 在 `ranging` 市场中应降低趋势信号的措辞强度。

## 分析原则

1. 先下结论：`decision.signal` + `strength` + `signal_status`
2. 验证主信号：`primary_signal` 触发来源，`bars_since_signal` 越小越新；检查 reasons 中是否有缩量金叉或 EMA60 提示
3. 校验辅助因子：统计 `confirming` / `conflicting` / `risk` / `opportunity` 数量
4. 给执行边界：`risk.stop_loss` / `target_1` / `target_2` / `trailing_stop` / `risk_reward_ratio` / `risk_level`
5. `focus` 才展开分析，`观察` 只统计不展开
